# Microservices vs Monolithic Architecture

## Monolithic Architecture
- Single codebase
- All components tightly coupled
- Hard to scale specific parts

### Advantages:
- Simple to develop
- Easy deployment (single unit)

### Disadvantages:
- Difficult to scale
- Hard to maintain for large apps
- Single point of failure

---

## Microservices Architecture
- Application divided into small services
- Each service runs independently
- Communicates via APIs

### Advantages:
- Independent deployment
- Better scalability
- Fault isolation

### Disadvantages:
- Complex setup
- Requires containerization (Docker)
- Needs orchestration tools

---

## Real-life Example:
- Netflix → Microservices
- Small college project → Monolithic
